◆ fur_coms_chars_t

enum TEF::FurComs::fur_coms_chars_t

Special FURCOMS characters.

This enum defines the special characters used in Furcoms encoding. Encoding is performed in the same manner as for SLIP encoding (RFC 1055), however with the END character modified to be 0x00. This ensures that END is a dominant character on a CAN-Bus line, and that it can thusly overwrite messages.

See also
https://tools.ietf.org/html/rfc1055

◆ handler_state_t

enum TEF::FurComs::handler_state_t

Transceiver bus state.

This enum defines the states that a bus transceiver can be in during any given moment in time, be it a idle bus, waiting for transmission of receiving data.

The following states are relevant for transmission only:

  • PARTICIPATING_ARBITRATION
  • SENDING
  • SENDING_COMPLETE

A listen-only node may leave these states out.

At any given time, if the bus has been idle for more than 5ms, any received 0x00 will count as a START and not packet STOP, and thusly trigger a new message. However, it is advised that nodes may only send a 0x00 after 10ms of bus idle time, to ensure that all nodes agree on the START condition.

Enumerator

Bus is idle, sending is permitted at any time.
